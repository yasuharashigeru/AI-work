## WebアプリをiPhoneのホーム画面アプリにする方法

**習得日：** 2026年7月第1週
**更新日：** 2026年7月4日（Netlify無料枠の上限到達につき、GitHub Pagesに移行）

### 概要

ClaudeCodeで作ったWebアプリ（HTMLファイル）を、GitHub Pagesで公開URLを取得し、iPhoneのSafariでホーム画面に追加することで、ネイティブアプリのように使える。

> 以前はNetlifyのドラッグ＆ドロップでデプロイしていたが、無料プランの上限に達したため、リポジトリ `AI-work`（`https://github.com/yasuharashigeru/AI-work`）をそのままGitHub Pagesで公開する方式に切り替えた。

---

### 登場するもの

| 名前 | 種類 | 役割 |
|------|------|------|
| GitHub Pages | Webホスティングサービス | リポジトリの中身をそのままインターネット上に公開する |
| Safari（iPhone） | ブラウザ | 公開URLを開き、ホーム画面に追加する |
| ホーム画面追加 | iOSの機能 | WebアプリをアプリのようにiPhoneに設置する |

---

### 手順

#### 1. アプリ用フォルダをリポジトリに追加する

1. `AI-work` リポジトリ直下にアプリごとのフォルダを作る（例：`Traqr/`）
2. その中にHTMLファイルと `icon.png` を置く
3. フォルダ内のメインファイルは **`index.html`** という名前にしておくと、URLがフォルダ名だけで済んで短くなる（例：`Traqr_AllData/index.html`）
4. `git add` → `git commit` → `git push` でリポジトリに反映

> ⚠️ GitHub Pagesは公開リポジトリの中身がそのまま誰でも見える状態になる。個人情報・機密情報を含むファイルを置く場合は事前に要検討。

#### 2. GitHub Pagesを有効化する（最初の1回だけ）

1. `https://github.com/ユーザー名/AI-work/settings/pages` を開く
2. **Build and deployment → Source** で「**Deploy from a branch**」を選択
3. **Branch** で「**main**」／「**/ (root)**」を選んで **Save**
4. 数十秒〜数分で `https://ユーザー名.github.io/AI-work/` 配下に公開される
5. 有効化済みなら「Your site is live at 〜」という表示が出るので、それで確認できる（Save済みで変更が無ければボタンがグレーアウトするのは正常）

> リポジトリ直下に空の `.nojekyll` ファイルを置いておくと、GitHubのJekyll変換処理をスキップしてファイルをそのまま配信できる。

#### 3. 各アプリのURLをiPhoneのSafariで開く

- URLの形式：`https://ユーザー名.github.io/リポジトリ名/フォルダ名/ファイル名.html`
  （`index.html`の場合はファイル名を省略してフォルダURLだけでOK）
- 画面下部の **共有ボタン**（四角から矢印が出るアイコン）をタップ
- **「ホーム画面に追加」** を選択
- 名前を確認して **「追加」** をタップ

→ ホーム画面にアイコンが現れ、タップするとSafariのUIなしで全画面で開く。ネイティブアプリと同じ見た目になる。

---

### アイコンについて

- ホーム画面のアイコンは各アプリフォルダに置いた **icon.png** が使われる（相対パス指定なのでフォルダ構成が変わっても壊れない）
- HTMLの `<head>` 内に以下を記述しておくとiPhoneが自動で認識する：

```html
<link rel="apple-touch-icon" href="icon.png">
```

- 推奨サイズ：**180×180px**（iPhoneのRetina対応）

---

### まとめ

```
ClaudeCodeでHTML作成
        ↓
AI-workリポジトリのアプリフォルダに配置してgit push
        ↓
GitHub Pagesで公開（最初の1回だけ設定）
        ↓
公開URLが発行される（https://ユーザー名.github.io/AI-work/フォルダ名/）
        ↓
iPhoneのSafariで開く
        ↓
ホーム画面に追加
        ↓
ネイティブアプリのように使える
```

## git push後、iPhoneに更新が反映されないときの対処

**習得日：** 2026年7月4日

### 症状

`git push`してGitHub Pagesの配信内容（サーバー側）は最新になっているのに、iPhoneのSafariで正規URL（`https://ユーザー名.github.io/...`）を開くと更新前の古い画面のまま。PCのブラウザやローカルサーバー経由（同一Wi-Fi内のPCのIPに直接アクセス）では問題なく最新版が見える。

### 原因

GitHub Pagesが使っているCDN（Fastly）の、たまたまiPhoneの通信が経由する中継地点だけに古いキャッシュが残ってしまうことがある（まれな現象）。以下を全部試しても直らないことがある：

- URL末尾に `?v=2` などを付けて別ページ扱いにする
- 10分以上待つ（GitHub Pagesの`Cache-Control: max-age=600`の期限切れを待つ）
- Wi-Fi⇔モバイル回線の切り替え
- iCloudプライベートリレーのOFF
- 設定 → Safari → 詳細 → Webサイトデータ → 該当ドメインを完全削除

→ これらはどれも**クライアント側**のキャッシュ対策なので、原因が**CDN側**にある場合は効かない。

### 切り分け方

同一Wi-Fi内であれば、PCでローカルサーバーを立てて（例：`python -m http.server 8791`）、iPhoneから `http://（PCのローカルIP）:8791/フォルダ名/` に直接アクセスすると、GitHub PagesのCDNを一切経由せずに「今この瞬間のファイルの中身」を確認できる。ここで正しく表示されれば、コードは問題なく、原因はCDN側と確定できる。

### 確実な直し方

ファイルの中身を実際に1行変える（`<head>`内にコメント `<!-- build: 日付 -->` を足すなど）→ コミット → push。中身が変わらない空コミット（`git commit --allow-empty`）ではファイルの`ETag`が変わらず効果がないので注意。中身を変えて`ETag`を新しくすると、CDNが強制的に新しい内容を取りに行くため、たいてい直る。

> ⚠️ HTMLファイルの中身を変える際、`<!DOCTYPE html>`より前にコメントなどを置くと、ブラウザが「標準準拠モード」に入らなくなる場合があるので、必ず`<head>`の中など`<!DOCTYPE html>`より後ろに置くこと。
