# Sake — notes for Claude Code

うまかったお酒を記録するPWA。iPhoneのホーム画面に追加して使う想定。Traqr/Calendarと同じ「vanilla JS・単一HTML・フレームワークなし・Meiryoフォント」の作風（詳細はObsidianの `Claude\Rule\` を参照）。仕様の原典はObsidianの `Claude\⚪︎App-Idea\酒管理ソフト.md`。

## これはWebアプリ（PWA）。ネイティブ機能は使っていない

`index.html`だけで完結する純粋なブラウザアプリ。`require('electron')`のようなものは一切なく、`file://`で直接開いても動く（ただしService Workerの登録とPWAインストールはhttps/localhostのようなsecure contextでないと機能しない）。

## データの持ち方

- IndexedDB（DB名 `sakelog`）に `genres`（ジャンル）と `items`（お酒の記録、写真はBlobで直接格納）を保存。**端末内のみ**で完結し、他端末との同期は一切ない。
- 写真は「最初にトリムした正方形1枚」だけを保存する（元画像は保持しない）。再トリムしたい場合は写真を選び直す必要がある — これは意図的な仕様（Obsidianの元アイデアメモの「さいしょにどうトリムするか調整できる」という書き方に基づく判断）。
- バックアップはJSON書き出し/読み込み（設定シートから）。写真はBase64化してJSON内に埋め込む。読み込みは全置換で、直前の状態はセッション中のみ有効なメモリ上のスナップショットから「元に戻す」で復元できる（ページを閉じると復元不可）。

## GitHub Pagesでのデプロイ（サブパス配信の注意）

このリポジトリ（`AI-work`）のGitHub Pagesはルート直下から配信されるため、実際のURLは `https://yasuharashigeru.github.io/AI-work/Sake/` のようにサブパスになる。そのため`manifest.json`・`service-worker.js`・アイコンの参照は**すべて相対パス**にしてある。絶対パス（`/xxx`から始まるもの）を書くとサブパス配信で壊れるので、今後手を入れるときも相対パスを維持すること。

Service Workerはアプリシェル（`index.html`, `manifest.json`, アイコン）をcache-firstでキャッシュしてオフライン起動を可能にしている。中身を更新したときは`service-worker.js`の`CACHE_NAME`をバージョンアップしないと、古いキャッシュが残って更新が反映されない端末が出る。

## アイコンについて

`icon-180.png` / `icon-192.png` / `icon-512.png` はPowerShellの`System.Drawing`で生成した簡易プレースホルダー（緑の角丸背景に「酒」の文字）。ちゃんとしたアイコンに差し替えたくなったら、同じ3サイズで置き換えるだけでよい。

## 検証方法

- ローカルでは `npx serve .`（このディレクトリで）のような静的サーバーを立てて、Chrome DevTools のモバイルエミュレーションでCRUD・クロップ座標計算・IndexedDBの永続化（Applicationタブ）を確認できる。
- ただしカメラ/フォトライブラリからの写真選択、クロップの実際の指の動き、ホーム画面追加後のアイコン表示などの**iOS Safari固有の挙動はChromiumのシミュレーションでは代表できない**。GitHub Pagesにデプロイした実URLを実機iPhoneのSafariで開いての手動確認を経てから「動作確認済み」と判断すること（Chromium自動テストのみでの断定は避ける）。

## 未実装・既知の制約（v1時点）

- 複数端末同期なし（意図的な仕様、[[Rule/データ保存先の判断]]参照）。
- ジャンル並び替えはドラッグ&ドロップではなく▲▼ボタン（タッチでのドラッグ実装を避けた簡易対応）。

## FB1対応（2026-07-15）でのUI再構築

実機フィードバックを受けて、入力フォームを角丸カードから罫線区切りのフラットデザインに再構築し、テーマカラーを緑(`#13A18C`)から水色(`#7FD4F5`)へ変更した。CSS変数は`--green*`系から`--accent*`系にリネーム済み。`--accent`（`#7FD4F5`)はボタン背景など大面積の塗りに、`--accent-ink`（`#0B4A66`、濃紺）はリンクや小さいテキスト・アイコンなど視認性が必要な箇所に使い分けている（パステル調の水色はそのまま小さい文字色にすると単体でコントラスト比が低すぎるため）。

写真は既存の1枚をタップするとアクションシート（トリム調整／新しい写真を選ぶ／削除）が開き、「再トリム」は保存済みの正方形Blobを`openCrop()`に再度通す形で実現している（元画像を持たない設計は維持したまま、CLAUDE.mdの上記「再トリム不可」という記述は解消された）。

## iPhone実機での「保存が押せない」問題と対策（FB2、2026-07-15）

保存/リセットボタンを`.form-actions`として`position:fixed`で下部ナビの直上に固定する実装を最初に試したが、**実機のSafariでは直らなかった**（ヘッドレスChromeでの`Emulation.setDeviceMetricsOverride`によるビューポート縮小シミュレーションでは問題を再現できていたつもりだったが、これはSafari本体のツールバー開閉の実際の挙動を模したものに過ぎず、真の動作を保証するものではなかった）。

最終的な対策：`body`を`display:flex; flex-direction:column; height:100dvh`のシェルにし、header/nav/保存バーを`position:fixed`ではなく通常のflexアイテム（`flex-shrink:0`）として配置、`<main>`だけを`flex:1; overflow-y:auto`のスクロール領域にした。さらに`100dvh`のCSS対応状況が実機のiOSバージョンによってはあてにならない可能性を考慮し、`window.visualViewport.height`を`resize`/`scroll`イベントで監視して`document.body.style.height`に反映するJSも追加し、CSSのdvh依存を実質的になくした。あわせて`window.addEventListener('error'/'unhandledrejection', ...)`でJSの例外をトースト表示するようにした（今回のように「タップしても何も起きない」系の不具合は、エラーが握りつぶされているだけなのか本当に無反応なのかを外から判別できないと診断できないため）。

**教訓：** `Emulation.setDeviceMetricsOverride`でビューポートサイズを人為的に変えるテストは、Safari本体の可変ツールバーやvisualViewportの実際の挙動を代表しない。この種の「実機のSafariでだけ起きる」系の不具合は、CDPでの検証結果だけで「直った」と断定せず、必ずユーザーの実機報告を待つこと。

## FB2対応（2026-07-15）でのUI変更

- ヘッダーの戻るボタンを`display:none`ではなく`visibility:hidden`で隠すようにし、非表示時もレイアウト幅を確保してタイトル文字が真ん中に来るようにした（同じ理由で右上のアクションボタンも`visibility`で切り替え）。
- 右上のアイコンは画面によって内容が変わる汎用ボタン`#headerAction`に統合：閲覧トップ（ジャンル一覧）ではハンバーガーメニュー（設定シートを開く）、ジャンル内のアイテム一覧では並び替えアイコン（投稿日・スコア・名称×昇順降順の`#sortSheetPanel`を開く）、それ以外の画面（入力タブ・詳細画面）では非表示。
- ジャンル管理をTraqrの「行をタップ＝選択、左にスワイプ＝名前変更/削除のアクションモーダル」パターンに合わせて再構築（ただし実際にnative `prompt()`/`confirm()`を使うTraqrとは異なり、Sakeは[[Rule/見た目にこだわるなら最初からネイティブUI要素を避ける]]の方針を維持するため、同じ見た目のカスタムモーダル（`.item-action-modal`）で代替した）。並び替え▲▼ボタンはTraqrには存在しない機能だが、Sake側で先に実装済みだったため維持し、各行の右端に残している。
- アイテムグリッドのサムネイルは長押し（550ms、`pointerdown`/`pointerup`/`pointerleave`/`pointercancel`で判定）で削除確認モーダルを出せるようにした。長押し発火後はクリックイベントを1回だけ無視して詳細画面への遷移を防いでいる。
- 「URL」フィールドを「製造元」（テキスト）と「リンク」（URL、任意）の2フィールドに分離。詳細画面では製造元名がリンクがあればクリック可能なリンクとして、なければ通常テキストとして表示される（**2026-07-16に方針転換、下記参照**）。

## FB2追加対応（2026-07-16）

- ジャンル管理の▲▼ボタンを廃止し、長押し(450ms)→縦ドラッグでの並び替えに変更（掴んでいる間は`.genre-row-dragging`でグレー表示、隣の行の中間点を超えると即座に入れ替わる）。スワイプでの削除/名前変更も、指の動きに`translateX`で追従してから発火するように変更。**重要：Traqr自体には並び替え機能もスワイプ追従アニメーションも存在しない**（コード確認済み）。ユーザーが「Traqr同様」と言っても実際のTraqrの挙動と食い違うことがあるので、憶測で実装せず必ずコードを確認すること。
- `img`/`.item-tile`/`.genre-tile`/`.photo-picker`に`-webkit-touch-callout: none; user-select: none;`を追加。iOSの長押しで出る「共有/写真に保存」の標準コンテキストメニューが、アプリ自前の長押しジェスチャーと競合するのを防ぐ（ヘッドレスChromeでは再現できないため実機確認待ち）。
- **入力欄フォーカス時に画面全体が上にずれる問題**：`main`だけがスクロールする設計にしていたが、`html`/`body`自体がposition:fixedでなかったため、iOS Safari側の「フォーカスした入力欄が見えるようスクロールする」処理がページ全体（bodyごと）を動かしてしまい、ヘッダー/ナビが上端に張り付いて下に空白ができる不具合があった。`html`・`body`両方に`position:fixed; width:100%`を追加し、ページ全体が動けないようにして`main`の内部スクロールだけで完結させるようにした。
- 「リンク」フィールドは「製造元のリンク」ではなく「購入リンク」（販売ページへのリンク、製造元名とは無関係の独立項目）だとフィードバックがあり、ラベルを「購入リンク」に変更し、製造元名からリンクを外して単なるテキスト表示に戻した。
- アイテムグリッドのサムネイルが名称の長さでサイズが変わる不具合の**真因はgrid item側の`min-width:auto`のデフォルト値**だった（`.item-tile-name`に`white-space:nowrap`を設定していても、CSS Gridは子要素のintrinsic minimum widthを尊重するため、折り返されないテキストがあるとグリッドトラック自体が広がってしまう）。`.item-tile-name`の高さを固定しただけでは直らず、`.item-tile { min-width: 0; }`を追加して初めて3等分が安定した。同様の「grid内の子要素サイズがおかしい」系の不具合が今後出たら、まずこの`min-width:auto`のデフォルトを疑うこと。
- 名称・製造元にそれぞれ「ふりがな（任意）」の入力欄を追加（`nameRuby`/`makerRuby`）。閲覧の詳細画面では名前・製造元とも普通の黒文字表示のまま（リンクがあるようには見せない）で、タップすると`showToast()`でふりがなを表示する仕組み。ふりがなが未入力ならタップしても何も起きない。
